## 1. Executive Summary

**Overall health.** The codebase has reasonable test infrastructure for its *pure, transport-agnostic* modules (protocol, transport, terminal emulator, model geometry, prompt, renderer, config) and weak-to-absent coverage for its *syscall/orchestration* modules (pty, input, client, main, commands, events). Coverage is high where it is easy (protocol 18/19, transport 2/2, prompt 15/17, renderer 3/3 public) and collapses where the code is load-bearing for real terminal behavior.

**The 3-5 biggest risks:**

1. **PTY tests are effectively ZERO coverage in CI.** All 6 tests in `pty-tests.lisp` self-skip via `(unless (pty-available-p) (skip ...))`, and the sanctioned Nix sandbox has no `/dev/ptmx`. The documented `pty-close` kill-broadcast guard (must-not-regress), `terminal-size` clamp, `disable-raw-mode!` NIL-guard, and `select-fds` empty-list guard all run zero assertions in CI â€” yet several are testable *without* a PTY.
2. **Most model/window-lifecycle tests also skip in CI.** 7 of 8 model-tests gate on `pty-available-p`; the recently-fixed (`44ae6de`) resize path â€” `window-relayout` / `ensure-window-fits` â€” has no direct test at all, despite being the exact regression-prone code. These are testable with the `make-pane :fd -1` no-PTY pattern but are not written that way.
3. **Modules with NO test file at all: commands, client, input, main** (and `events.lisp` has no formal exports). `commands` (`resize-pane` and its geometry helpers â€” ~half the module) is only reached transitively through events dispatch. `client`/`input`/`main` are untested entirely.
4. **The thin `server-tests`** plus the structural fact that **`test/e2e-smoke.lisp` exists on disk but is NOT registered in `cl-tmux.asd`'s test components** (ASD lines 53-64 list 12 `*-tests` files; `e2e-smoke` is not among them) â€” so it never runs. Dead test file.
5. **Copy-mode escape-sequence byte machine in events is untested end-to-end.** `%process-escape-byte` + the ESC-arming branches (the byte-at-a-time arrow-key flow that production actually hits) are the most intricate stateful logic in events and have no test, though they are unit-testable today.

**Headline structural findings:**
- Skip-gating masks the true CI coverage: the only environment that can run tests (Nix sandbox) silently skips the majority of pty/model tests.
- Mis-suited tests: several `pty-tests` actually assert model/commands behavior; `commands` units are tested via `events`; `prompt-tests` spans 3 modules (prompt + events + renderer).
- Several exported public units have no caller in their own module: terminal's `cursor-up/down/left/right` (CSI routes through `set-cursor` instead) â€” dead public API or a latent scroll-region bug.

## 3. Prioritized Gap List (high â†’ low)

### HIGH
- **pty / `pty-close`** â€” the `(child-pid > 0)` / `(master-fd >= 0)` guard preventing `kill(-1)`/`kill(0)` broadcast to the whole process group (including the test runner). Flagged must-not-regress; testable with NO PTY.
- **pty / `terminal-size`** â€” three branches (ioctl success, 1..1000 sanity clamp, 24Ă—80 fallback) all untested; a transient 0Ă—0 read silently corrupts layout. Runnable in the sandbox.
- **pty / `enable-raw-mode!` & `disable-raw-mode!`** â€” raw-mode entry/restore; a missed restore leaves the terminal cooked. The `disable-raw-mode!` NIL-guard is testable; `enable-raw-mode!` needs a real tty.
- **terminal / `cursor-up/down/left/right`** â€” four exported movement primitives with NO caller and NO test; CSI routes through `set-cursor` instead. They clamp to the **scroll region**, not screen bounds â€” a divergence tests cannot currently see.
- **terminal / DCH+ICH (`delete-chars` / `insert-chars`, CSI `P`/`@`)** â€” non-trivial shift loops with off-by-one-prone bounds `(- w n 1)`, `(max cx (- w n))`; zero tests at any layer.
- **model / `window-relayout`** â€” core SIGWINCH-resize path for split windows; both the NIL-layout fallback and the split-preserving branch are unasserted. Exactly the code `44ae6de` just fixed.
- **model / `ensure-window-fits`** â€” both branches untested: relayout-on-change and the no-op-when-equal perf/correctness invariant.
- **commands / `resize-pane` + `%resize-vertical/horizontal`** â€” ~half the module; split-orientation ecase, neighbour selection, min-size>2 guard, `+1` separator arithmetic. Pure geometry, no PTY needed.
- **events / `%process-escape-byte` + ESC-arming-in-copy-mode** â€” the hand-rolled byte-at-a-time arrow-key state machine; production hits this on every arrow key, tests never do.

### MEDIUM
- **net / `socket-fd`** â€” sole outright-untested export; the server event loop registers sockets with `select-fds` via it.
- **protocol / large payload (â‰Ą256, â‰Ą65536)** â€” all tested payloads â‰¤6 bytes, so the upper u32 length bytes are never exercised; `msg-frame` carries whole rendered screens in production.
- **transport / truncated payload (full header, short body)** â€” the realistic mid-frame peer-hangup path; only header-truncation is tested.
- **config / `%command-keyword`** â€” the `find-symbol` non-interning guarantee and the `*bindable-commands*` member gate are asserted only indirectly; a refactor to `INTERN` or dropping the member check would pass all current tests.
- **renderer / active-prompt branch** â€” `render-status-bar` replacing the left segment with `(prompt-text)` is never tested (only the inactive branch).
- **renderer / border clamps** â€” horizontal `pw` clamp and vertical off-screen suppression never exercised (test fixtures always keep the border on-screen).
- **commands / `copy-mode-scroll`** â€” invoked but never asserted: the `[0, len scrollback]` clamp, the dirty side-effect, and the not-in-copy-mode no-op.
- **commands / `kill-pane` multi-pane survivor + empty-windowâ†’kill-window** â€” only the single-pane `:quit` path is hit.
- **events** â€” `handle-prompt-key` Enter/Esc/Backspace branches; `:prev-pane`, `:resize-*` dispatch arms; prefix `[`/`]` copy-mode interception and fall-through.
- **client** â€” frame-dispatch cond (`+msg-frame+` decode-text, `+msg-bye+`/null â†’ exit); socket cleanup on error path.
- **input** â€” no test file; `read-byte-nonblock` timeout/EOF-conflation; `with-raw-mode` double-disable ordering.

### LOW
- terminal: ECH/SU/SD, `dec-pm-reset` fallback, scrollback 1000-cap, `cursor-bs` col-0 guard, `cursor-ht` clamp, `safe-code-char` U+FFFD, charset/keypad/DSR/DA no-ops, `screen-dirty-p`, `clamp` direct cases.
- config: `config-file-path` wrapper, `+pty-buf-size+` value-pin, tokenizer tab/whitespace, indented comments, set/remove return values.
- prompt: empty-buffer `prompt-text` spacing, multibyte input, `overlay-lines` trailing newline.
- protocol: direct `read-u32` byte-order, zero-length subseq path, malformed-UTF-8 `decode-text`.
- net: explicit `:backlog`, double-close idempotency, probe-path cleanup.
- main: argv dispatch (`server`/`attach`/default + `"0"` name fallback).

## 4. Recommended New Tests

### HIGH severity first

**pty (sandbox-OK â€” converts ~0 effective CI assertions into real coverage):**

- `pty-close-ignores-non-positive-pid` â€” protects the kill-broadcast invariant:
  ```lisp

  ```


## 5. Verification Note

These tests cannot be run with a native SBCL â€” it **hangs loading cffi**. The only sanctioned way to build and run the suite is the sandboxed Nix flake check:

```
git add -A
nix build .#checks.aarch64-darwin.default -L
```

`git add -A` first is mandatory: **flakes only see git-tracked files**, so any new test file or ASD edit is invisible until staged. (slow build; expect the run to take a while.)

Two CI realities to keep in mind when adding tests:
- The sandbox has no `/dev/ptmx`, no controlling tty, and (per `unix-socket-available-p`) may forbid Unix-socket bind. Any test gated on `pty-available-p` / `unix-socket-available-p` will **SKIP**, not fail â€” so prefer the no-PTY (`make-pane :fd -1`, direct `make-screen`/`make-input-state`) patterns above to get real assertions executing in CI.
- **`test/e2e-smoke.lisp` is not registered** in `cl-tmux.asd` (test components are ASD lines 53-64; e2e-smoke is absent) and is not in `test/suite.lisp`, so it never runs regardless. Either wire it in or remove it.

## Addendum â€” server & runtime (recovered manually)

The propose-stage agents for `server` and `runtime` finished without emitting structured output, so both items dropped from the auto-synthesis above. Filled in by hand from the source:

**server.lisp** (4 internal units; no `defpackage` exports â€” `in-package #:cl-tmux`):
- `socket-path` â€” **covered** (`server-tests::socket-path-includes-session-name`).
- `apply-client-size` â€” **covered** (`server-tests::apply-client-size-updates-dimensions-and-dirties`).
- `serve-client` â€” **GAP (medium).** The `cond` over frame types (null/disconnect, `+msg-detach+`, attach/resize â†’ `apply-client-size`, `+msg-key+` â†’ `process-byte` loop with `:quit` â†’ sets `*running* nil`/`outcome :quit`) plus the `unwind-protect` `msg-bye`+`close-socket` cleanup are never asserted. *Recommendation:* extract a pure `(server-handle-frame session state type payload) â†’ {:quit | :detach | nil}` helper (mirrors the suggested `client-handle-frame` refactor), then unit-test each branch â€” especially that a `:quit` from `process-byte` propagates to `outcome`. The full accept/serve loop with a live socket remains infeasible in the sandbox.
- `run-server` â€” **GAP (low / needs infra).** Bind/accept/reader-thread loop; integration-only, needs a live socket + PTYs.

**runtime.lisp** (2 functions + 5 specials; `in-package #:cl-tmux`):
- `install-sigwinch-handler` â€” **GAP (medium).** Installs a real `sb-sys:enable-interrupt` handler whose lambda sets `*resize-pending*`/`*dirty*`. *Recommendation:* extract the body into a pure `(%on-sigwinch)` that flips the two specials and assert it under `let`-rebound flags â€” testing the actual signal delivery is fragile and not worth it.
- `start-reader-thread` â€” **GAP (low / needs infra).** Spawns a thread doing blocking `pty-read-blocking`; needs a live PTY, will self-skip.
- specials (`*dirty* *running* *resize-pending* *term-rows* *term-cols*`) â€” state; no direct test warranted (already exercised indirectly by events/server tests via `let`-rebinding).
