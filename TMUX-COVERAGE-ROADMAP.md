# cl-tmux Implementation Roadmap: 100% tmux Feature Coverage

## 1. Executive Summary

**Current implementation coverage: approximately 12–15%** of tmux's full feature surface.

Implemented and working:
- Single session with multiple windows and panes (binary split tree)
- Horizontal/vertical pane splitting (C-b % and C-b ")
- Basic key bindings via `*key-bindings*` hash table
- Terminal emulator core: SGR 0–7/22–27, CSI cursor movement, alternate screen
- Config file loading with `set-shell` and `set-status-height` directives
- Unix socket client/server with message framing
- Copy mode (scroll only, no selection)
- Status bar (hard-coded format)
- PTY integration via cffi

**Top 5 blockers to usability:**

1. **Single-session server** — the entire multi-session model is absent; `run-server` creates exactly one session and exits when it dies. Nothing else in tmux works without this foundation.
2. **256-color / true-color SGR** — virtually all modern terminal applications (neovim, lazygit, etc.) emit `38;5;n` or `38;2;r;g;b`; without it cells render with wrong colors or garbage.
3. **Session targeting (-t flag)** — no command can address a specific session/window/pane, making scripting and multi-session use impossible.
4. **Format-string expansion (#{var})** — the status bar, window titles, and dozens of commands depend on this; without it `status-left`, `status-right`, and `set-option` values are inert strings.
5. **Directional pane navigation (select-pane -L/-R/-U/-D)** — C-b arrow keys are unbound; users are stuck with C-b o cycling, which is unusable beyond 3 panes.

### S0-2: 256-color and true-color SGR

**What:** `apply-sgr` in `src/terminal/sgr.lisp` processes a flat parameter list with single-value dispatch. It cannot handle `38;5;n` (3 params) or `38;2;r;g;b` (5 params).

**Files:** `src/terminal/screen.lisp`, `src/terminal/sgr.lisp`, `src/renderer.lisp`

**Changes:**

In `src/terminal/screen.lisp`:
- Change `cell` struct fields `fg` and `bg` from `(unsigned-byte 8)` to a new type. Define:
  ```lisp
  ;; color representation
  (defstruct color
    (kind :default :type (member :default :ansi :256 :rgb))
    (value 0 :type (unsigned-byte 32)))  ; ANSI index, 256 index, or #xRRGGBB
  ```
  Or use a simple fixnum encoding: 0 = default, 1–16 = ANSI, 17–272 = 256-color (value-16), high bits = RGB flag.
  Recommended: use a `(or null (unsigned-byte 25))` where bit 24 = RGB flag, bits 16–23 = R, 8–15 = G, 0–7 = B; values 0–255 = palette index; NIL = default.

In `src/terminal/sgr.lisp`:
- Replace the `define-sgr-rules` entries for parameters 38 and 48 with a multi-parameter handler.
- Add a new function `parse-extended-color (params index)` → `(values color next-index)` that handles:
  - `(38 5 n)` → 256-color fg, consume 3 params
  - `(38 2 r g b)` → true-color fg, consume 5 params
  - `(48 5 n)` → 256-color bg
  - `(48 2 r g b)` → true-color bg
- Change `apply-sgr` from iterating `(dolist (p params))` to a loop with an index, calling `parse-extended-color` when `p` is 38 or 48.

In `src/renderer.lisp`:
- Update `render-cell-fg` / `render-cell-bg` to emit the correct escape sequence based on the color kind:
  - `:ansi` (0–7): `\e[3Nm` / `\e[4Nm`
  - `:ansi` (8–15): `\e[9Nm` / `\e[10Nm`
  - `:256`: `\e[38;5;Nm` / `\e[48;5;Nm`
  - `:rgb`: `\e[38;2;R;G;Bm` / `\e[48;2;R;G;Bm`

**Tests:** `test/unit/renderer-tests.lisp` — add `(test 256-color-sgr ...)` and `(test true-color-sgr ...)`.

### S1-5: Pane zoom toggle (resize-pane -Z, C-b z)

**Changes to `src/pane.lisp`:**
- No struct change needed; zoom is a window-level concept.

**New function in `src/dispatch.lisp`:**
```lisp
(defun cmd-zoom-toggle (server)
  (let* ((win (current-window server))
         (pane (window-active-pane win)))
    (if (window-zoomed win)
      (cmd-zoom-out win)
      (cmd-zoom-in win pane))))

(defun cmd-zoom-in (window pane)
  (setf (window-zoom-saved-layout window) (window-layout window))
  (setf (window-layout window) (make-leaf-layout pane (window-rect window)))
  (setf (window-zoomed window) t)
  (reflow-window window))

(defun cmd-zoom-out (window)
  (setf (window-layout window) (window-zoom-saved-layout window))
  (setf (window-zoom-saved-layout window) nil)
  (setf (window-zoomed window) nil)
  (reflow-window window))
```

**Changes to `src/input.lisp`:**
- Bind C-b z → `:zoom-toggle`.

**Changes to `src/renderer.lisp`:**
- When `(window-zoomed window)` is T, `render-window` should render only the active pane at full window dimensions, ignoring the layout tree.

## 5. Sprint 3 — P3 Completeness Features

### Session groups and advanced session management
- **Session groups** (`new-session -t`): share windows across sessions; requires `session-windows` to hold references, not owned objects; `session-group` struct linking sessions.
- **lock-session / lock-client**: overlay a lock screen in renderer; accept passphrase via command prompt.
- **choose-session (C-b s)**: interactive TUI overlay listing sessions; navigate with j/k; Enter to switch. Reuse the `choose-window` overlay infrastructure from Sprint 2.
- **update-environment**: on session create, copy specified env vars from client environment into server-spawned PTY environments.

### Advanced window/pane operations
- **link-window / unlink-window**: shared-window mechanism (similar to session groups but at window level).
- **break-pane (C-b !)**: remove pane from current window's layout tree; create a new window containing it as sole pane. Modify `layout.lisp` `remove-pane-from-tree`; create window via `cmd-new-window`.
- **join-pane / move-pane**: inverse of break-pane; take a pane from another window and insert into current window's layout tree.
- **pipe-pane**: open a subprocess and tee all PTY output to it; add `pane-pipe-fd` slot; modify PTY read loop in `pty.lisp`.
- **synchronize-panes** window option: when enabled, key input to active pane is also written to all other panes in the window.

### Built-in named layouts
- **even-horizontal, even-vertical, main-horizontal, main-vertical, tiled**: add `cmd-select-layout` in `dispatch.lisp`; each layout algorithm takes the pane list and window rect and calls into `layout.lisp` to rebuild the tree with equal divisions.
- **select-layout and C-b Space / C-b M-1–M-5**: bind keys; `cmd-select-layout` cycles through or jumps to a named layout.
- **layout-persistence (layout string)**: implement `layout->string` and `string->layout` in `layout.lisp` following tmux's `WxH,X,Y[,...]` checksum format; needed for `display-panes` and session save/restore.

### Mouse support
- **Mouse reporting modes (?1000h/1002h/1003h/1006h)**: add to DEC mode table in `modes.lisp`; `screen-mouse-mode` slot. Input loop in `input.lisp` must parse incoming `\e[M` / `\e[<` sequences from the outer terminal.
- **Mouse pane selection**: on `MouseDown1Pane`, call `pane-at-position` (new function in `layout-geometry.lisp`) to find which pane contains the click coordinates; set active pane.
- **Mouse pane border resize**: detect click on border cell (between panes); enter drag mode tracking subsequent mouse-move events; update split ratio in layout tree.
- **Mouse wheel scrollback**: translate mouse wheel events to copy-mode scroll-up/down.
- **Mouse text selection**: enter copy mode on drag; update copy cursor; yank on button release.
- **Mouse status bar click**: map click column to window index in status bar; call `:select-window`.
- All mouse features should be gated behind `(get-option session "mouse")`.

### Scripting and hooks
- **hooks system**: add `*hooks*` hash table in new file `src/hooks.lisp` mapping hook names to lists of command strings; define hook names: `after-new-window`, `pane-exited`, `session-created`, `client-attached`, `client-detached`, etc.; `run-hook` iterates and calls `run-shell` on each.
- **run-shell**: add `cmd-run-shell` in `dispatch.lisp`; uses `sb-ext:run-program` or equivalent; captures stdout; returns to client.
- **if-shell**: `cmd-if-shell` runs a shell command and branches on exit code.
- **confirm-before**: wraps another command with a prompt overlay (reuse command-prompt); only dispatches the wrapped command on `y` input.
- **wait-for**: implement channel synchronization via a `*wait-channels*` hash in `src/runtime.lisp`; `wait-for foo` blocks until `wait-for -S foo` signals.
- **display-popup**: floating window overlay; requires a new rendering layer in `renderer.lisp` above the main grid; the popup has its own PTY and screen struct.
- **display-menu**: similar to display-popup but renders a text menu of labeled commands; navigation with j/k/Enter.
- **key-tables**: complete the key-table system started in Sprint 2; add `copy-mode` and `prefix` tables with full tmux-compatible bindings.

### Control mode and advanced client/server
- **control mode (tmux -C)**: line-oriented protocol where each notification is a `%begin`/`%end`-delimited block; requires a parallel `format-control-response` function in `protocol.lisp`.
- **concurrent multi-client**: server's event loop must support multiple simultaneous connected client file descriptors; each client gets its own rendered frame diff. Requires moving from single-fd `accept-connection` to a `select`/`epoll` loop in `server.lisp`.
- **command protocol over socket**: allow `tmux new-window`, `tmux send-keys`, etc. over an existing socket without a TTY; handle in `client.lisp` with a non-interactive `send-command` path.
- **read-only client**: track `client-readonly` flag; server skips input forwarding for such clients.

### Additional terminal emulation
- **Line drawing / ACS**: complete ESC `(0` / ESC `(B` character set switching already partially in parser; map ACS code points to Unicode box-drawing characters in `screen.lisp`.
- **DCS sequences**: parse in `parser.lisp` state machine; initially handle `DCS` pass-through for Sixel detection (can be a no-op that consumes the sequence).
- **Device Attributes (DA1/DA2)**: complete the existing partial implementation in `parser.lisp`; respond with `\e[?1;2c` (VT100 with AVO).
- **SGR double-underline (21), rapid-blink (26), underline-color (58/59)**: add bits to cell struct; extend `apply-sgr` and renderer.

## 6. Architecture Notes — Cross-Cutting Concerns

### Mouse support (Sprint 3) touches 6+ files
Mouse is the most invasive feature. Changes required in:
- `src/terminal/modes.lisp` — DEC modes 1000/1002/1003/1006
- `src/terminal/screen.lisp` — `screen-mouse-mode` slot
- `src/input.lisp` — parse `\e[M` / `\e[<N;C;Rm` sequences from outer terminal; decode into `(mouse-event button col row action)`
- `src/events.lisp` — new `mouse-event` struct; dispatch to pane/border/status-bar hit tests
- `src/layout-geometry.lisp` — new `pane-at-position (window col row)` function
- `src/renderer.lisp` — mouse mode tracking: enable ?1006h on outer terminal when any pane has mouse mode active
- `src/config-directives.lisp` / `src/options.lisp` — `mouse` boolean option

### Format strings (Sprint 1-7) touches renderer, config, and all command output
`src/format.lisp` must be loaded before `src/renderer.lisp`, `src/config-directives.lisp`, and any command that returns formatted output. Add it to `cl-tmux.asd` component list before `renderer`. The `format-context-from-session` function will need access to `src/session.lisp`, `src/window.lisp`, and `src/pane.lisp` — ensure those are listed before `src/format.lisp` in the `.asd` `:serial t` component list.

### Multi-session server requires protocol version bump
When sessions list is added to the server struct, the Unix socket protocol messages in `src/protocol.lisp` must be extended. Consider adding a `protocol-version` field to the handshake message (client capability negotiation, S1-9 prerequisite). All message types that carry a target must carry the `:target` string rather than assuming the single session.

### Key-table system (Sprint 2 config, Sprint 3 scripting)
Currently `*key-bindings*` is a flat hash table from key-chord to keyword. To support `-n` (no-prefix), `-T table`, and `-r` (repeatable), restructure as:
```lisp
(defvar *key-tables* (make-hash-table :test #'equal))
;; populated as: (gethash "prefix" *key-tables*) → hash of chord → (keyword . &options)
;;               (gethash "root"   *key-tables*) → hash of chord → (keyword . &options)
;;               (gethash "copy-mode" *key-tables*) → ...
```
The input dispatch in `src/input.lisp` must track a `*current-key-table*` dynamic variable, defaulting to `"root"`, switching to `"prefix"` on the prefix key, and to `"copy-mode"` when a pane is in copy mode.

### Colors require asd component ordering
The color type change in `src/terminal/screen.lisp` (Sprint 0-2) will break `src/terminal/sgr.lisp`, `src/renderer.lisp`, and `src/renderer-pane.lisp` if they are compiled before the new color struct is defined. `screen.lisp` is already first in the terminal subsystem ordering — verify the `.asd` `:depends-on` or `:serial t` ordering is `screen → cursor → modes → sgr → csi → parser`.

### Binary split tree must support zoom invariant
The `layout.lisp` zoom-in operation (Sprint 1-5) replaces the window's layout tree with a single-leaf tree. All functions in `layout-geometry.lisp` that walk the tree (including the new `pane-neighbor`) must handle single-leaf trees without error. Add a guard: if `(window-zoomed win)` is T, `pane-neighbor` returns NIL (no neighbors when zoomed).

### Renderer purity and incremental redraw
`render-session-to-string` is documented as pure. As features like zoom overlays, copy-mode highlighting, display-panes numbering, and popup windows add stateful overlays, keep all overlay state in the model (window/pane structs) rather than in the renderer. The renderer reads state; it does not mutate it.

## 7. Testing Strategy

### Sprint 0 tests
All Sprint 0 tests go in existing unit test files. Run via `nix build .#checks`.

- `test/unit/renderer-tests.lisp`:
  - `(test sgr-italic-bit)` — apply SGR 3 to a fresh cell; assert `cell-italic` is T, `cell-dim` is NIL.
  - `(test sgr-256-color-fg)` — parse `"38;5;200"` param list; assert cell fg encodes palette index 200.
  - `(test sgr-truecolor-fg)` — parse `"38;2;255;128;0"` param list; assert cell fg encodes RGB #FF8000.
  - `(test cursor-visibility-dec-mode)` — send `\e[?25l` through emulator; assert `screen-cursor-visible` is NIL.

### Registering new test files
Every new `test/unit/foo-tests.lisp` requires two changes:
1. In `cl-tmux.asd`, add under `:cl-tmux/tests` component:
   ```lisp
   (:file "unit/foo-tests")
   ```
2. In `test/suite.lisp`, add:
   ```lisp
   (fiveam:def-suite cl-tmux.tests/foo :in cl-tmux.tests)
   ```
   and verify the file's `(in-suite cl-tmux.tests/foo)` matches.

Run the full test suite after each sprint with:
```
nix build .#checks
```
Do not use native SBCL directly — cffi PTY bindings hang outside the Nix sandbox environment.
